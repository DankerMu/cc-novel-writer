# CS1 测试清单 — Quick Start Polish & Docs

**版本**：0.2.0
**前置**：安装 CLI，确认版本 0.2.0

---

## 测试 1: 冷启动创建项目

**操作**：在空目录下运行 `/novel:start`

**验证**：

- [ ] 是否提示「创建新项目」选项？
- [ ] 选择后是否一轮问完题材/主角/冲突？（不是反复追问）
- [ ] 题材是否有选项可选（玄幻/都市/科幻/历史/悬疑）？

**反馈**：截图或贴出交互过程

---

## 测试 2: 风格来源选择

**操作**：接上一步，进入风格来源选择

**验证**：

- [ ] 是否展示 4 个选项（原创样本/参考作者/预置模板/先写后提）？
- [ ] 选「先写后提」后是否跳过样本收集、直接进入世界观构建？

**反馈**：你选了哪个选项，后续行为是否符合预期

---

## 测试 3: WorldBuilder 轻量模式

**操作**：观察 WorldBuilder 输出

**验证**：

- [ ] `world/rules.json` 是否 ≤3 条规则？
- [ ] 规则是否全部标记 `hard`？
- [ ] 叙述文档（geography/history/rules.md）是否精简（≤300 字左右）？
- [ ] `storylines/storylines.json` 是否仅 1 条 `main_arc`？

**反馈**：贴出 `rules.json` 内容 + 文档字数估计

---

## 测试 4: 试写 3 章

**操作**：等待系统完成 3 章试写流水线

**验证**：

- [ ] 是否走了完整流水线（ChapterWriter → Summarizer → StyleRefiner → QualityJudge）？
- [ ] 完成后是否展示 3 章标题 + 字数 + 评分？
- [ ] 是否给出 3 个选项（进入卷规划/调整风格/重新试写）？

**反馈**：评分多少，3 个选项是否都显示

---

## 测试 5: 中断恢复

**操作**：

1. 在另一个空目录重新跑 `/novel:start`
2. 走到世界观构建完成后（Step D 之后），**手动中断**（关终端或 Ctrl+C）
3. 重新进入该目录，运行 `/novel:start`

**验证**：

- [ ] 是否检测到 `QUICK_START` 状态？
- [ ] 是否从中断处继续（不重新问题材/风格，不重建世界观）？
- [ ] 已生成的文件（world/、characters/）是否保留？

**反馈**：中断时机 + 恢复后从哪一步继续

---

## 测试 6: 用户文档可读性

**操作**：打开 `docs/user/quick-start.md`，按文档步骤比对你的实际体验

**验证**：

- [ ] 文档步骤与实际流程是否一致？
- [ ] 有没有看不懂或描述不准的地方？
- [ ] FAQ 是否有帮助？

**反馈**：哪些地方需要补充或修改

---

## 反馈模板

每个测试跑完后，按以下格式反馈：

```
测试 X: [通过/部分通过/失败]
问题：（如有）
截图/日志：（关键输出）
```

可以一次跑完所有测试再汇总反馈，不用逐个等。
